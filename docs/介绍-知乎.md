# 我给暖通工程师做了一个在线工具站：从热泵动态仿真到 CoolProp 高精度物性

> 一个不用装软件、打开浏览器就能算的暖通工具集。地址：https://hvac.geopro.cc

## 起因：暖通工程师的日常，卡在一堆零碎计算上

做暖通的人都熟悉这种场景：

- 查个冷媒在某温度下的焓值，翻物性表、插值，或者打开笨重的软件；
- 画个压焓图理解循环，手头没有趁手的工具；
- 湿空气状态换算，记不清 ASHRAE 那套公式，Excel 表格传了好几代；
- 水力、风管、能耗，各有各的算法，散落在不同地方。

这些计算单个都不难，但**零碎、重复、还容易记错单位**。我想要的其实很简单：一个网页，打开就能算，结果可信,单位随手切。于是有了这个 **HVAC 工具站**。

## 它能做什么

九个工具，一个入口：

| 工具 | 解决什么 |
|---|---|
| **系统仿真** | 变频一拖一热泵的动态可视化（制冷/制热/化霜/回油），拖参数看循环怎么变 |
| **冷媒物性** | 12 种冷媒任意状态点的焓/熵/密度/干度/黏度…（CoolProp） |
| **压焓计算** | 单级循环 COP/排温/能力 + **可拖拽的压焓图** |
| **湿空气** | 干湿球/RH/露点/含湿量任意两参数互算 + 焓湿图 |
| **水力计算** | 分段管阻→泵扬程→输送能效 EC(H)R 判定（含乙二醇） |
| **风管设计** | 等摩阻法选型 + 静压校核 + 自动画示意图 |
| **能耗电费** | 北美 SEER2/HSPF2 全年电费、双机型对比 |

12 种冷媒覆盖 R410A / R32 / R454B / R134a / R290 / R22 / R1234yf / R717(氨) / R744(CO₂) 等主流与新型工质。

## 几个我自己觉得值得说的点

### 1. 物性不是"拟合个近似公式"，而是直接调 CoolProp

市面上很多在线小工具，冷媒物性是用几个拟合系数糊出来的，边界一到就飘。这里所有物性计算走 **[CoolProp](http://www.coolprop.org/) 8.x** ——开源、被学术界和工业界广泛验证的物性库，`PropsSI` / `HAPropsSI` 直接算。

焓熵基准统一到 **IIR 标准**（0°C 饱和液 h=200 kJ/kg），和你手边的物性表册对得上，不用换算。R454B 这种预定义混合物查不到临界参数时，用 Opteon XL41 的文献常数兜底，两相区还会报泡露点和温度滑移——**近共沸/非共沸的滑移是很多工具直接忽略的**。

### 2. 压焓图可以"拖"

理解制冷循环，压焓图是最直观的。这里的压焓图不是一张静态图：**4 个状态点可以拖**——拖点改蒸发/冷凝压力、过热度、过冷度，松手自动 CoolProp 精算，鼠标悬停直接读 (h, P, 饱和温度)。想搞懂"过冷度加大对 COP 的影响",拖一下比讲十遍都清楚。

### 3. 一个会动的热泵

`系统仿真`页是我花力气最多的地方：变频压缩机、四通阀换向、EXV 开度、内外风机、结霜化霜、回油……冷媒粒子按**相态用不同形状和颜色**流动，流速正比于质量流量。拖动 9 个参数（转速、风量、EXV、冷媒量、换热器脏堵、内外温度），循环、测点、COP 实时响应，还带诊断提示（过热过低回液风险、排温过高等）。

它是**教学定性模型**——趋势对，但不当精确设计工具用。目的是让人"看见"热泵在干什么。配套还把节流前后、蒸发器进出口的冷媒状态（含节流后干度）都做成了测点。

### 4. 公制/美制，输入输出各切各的

做北美项目的都懂这个痛：图纸美制、习惯公制。这里**「输入制」和「输出制」是两个独立开关**——可以输入端用公制、输出端看美制，温度/压力/流量/面积全站跟着换算，选择还记忆。

## 技术选型：为什么是"无框架 + Python 标准库"

有人可能好奇实现。我刻意选了最"轻"的路线：

- **前端**：原生 HTML/CSS/JavaScript，Canvas/SVG 画图，**零构建、零框架**。打开即用，维护简单，十年后还能跑。
- **后端**：Python **标准库** `http.server` 一个文件，静态托管 + 一组 CoolProp JSON API。没有 Django/Flask，因为根本不需要。
- **认证**：Supabase（前端直连，publishable key）。

这种"够用就好"的克制，换来的是**极低的维护成本和极高的可移植性**。工程工具，稳定和可信比花哨重要。

## 已经上线，也踩了国内服务器的坑

站点部署在国内云服务器上，`https://hvac.geopro.cc` 和 `https://hvac.geotoday.net` 双域名 HTTPS。

部署过程里遇到一个很典型的坑：**国内服务器出境被限，连不上 Let's Encrypt 签不了证书**。最后的解法是——证书在能出境的本机用 acme.sh + 域名商 API 签发，再自动推送到服务器，配上定时任务自动续期。这套"本机中转"的思路对国内建站很有参考价值（细节我写在 CSDN 那篇里）。
