# 空调机队 FDD 项目方案 v1.1-Claude Fable5

报告时间：2026-07-02

基于 2026-07 全部讨论 + 2026-07-02 Zoho 标注流水线实测输出。v1.0 的结论性文档,不是可行性报告。

## v1.0 → v1.1 变更记录

1. **压力口径**:传感器确认为密封式表压 → 全机队固定 +1.013 bar 转绝压,压力换算**无需**海拔修正(v1.0 的海拔修正条目作废);海拔仅保留为风侧空气密度协变量。
2. **Sh 制热定性改写**:成熟稳态 Sh = 0.02±0.3 K(样本实测),EEV 将吸气过热伺服于近零目标,负值是零点附近噪声,不是数据缺陷;叠加测点位置偏置(工程经验:常态可到 −1~−2 K)后单机不可分、且设计不变。**Sh 降为 EEV 饱和后的晚期指标;制热泄漏主通道 = 同工况 Exv 漂移 + Sc + 容量残差。** 跨机 Sh 基线预期散布(约 −2~+1 K):单机自漂 = 事件(传感器漂移/充注变化),机间互异 = 良性。
3. **制热 Sc 偏置**:Tl 在室外机液管,制热时读数含室内冷凝器→室外机的管路温降(管长随安装而异)→ 制热 Sc 带安装依赖常值偏置,跨机绝对比较降权,逐机漂移检测不受影响。
4. **状态码落锚**:AcState 全表(1 正常停机 / 2 异常停机 / 3 等待运行 / 4 制冷 / 5 制热 / 6 回油 / 7 化霜 / 15 紧急制热 / 16 送风);St = 四通阀状态(1 制热位,0 化霜/制冷位);Tes/Tcs = 目标蒸发/冷凝温度,**Tcs 动态可变(样本内 38/39/40/45)→ "学习漂移 vs 故障漂移"获得直读判据:目标变 = 控制决策,追不上目标或追上但 Rps/Exv 代价抬升 = 能力短缺**。CompState 0/1/2 中 2 的语义收窄为一个固件问题(附录 A)。
5. **能效残差禁用 PowerIn**(含曲轴加热带),用 PowerComp(+风机);PowerIn − PowerComp − 风机 = 寄生功率基线(曲轴加热带失效可查)。Comp(0–50)/Fan(0–10)为指令,CompRps/FanRpm 为实际 → 电机/驱动诊断的指令-反馈对落锚。
6. **标签工程实测**:Zoho 标注流水线已于 2026-07-02 运行(11,772 行),各类底盘见 §4,数据质量问题与修复见 §5 阶段 1。
7. **待确认清单重排**:B-1 → 平台数据请求规格(附录 B);B-2 → 固件问题文本(附录 A);B-3 Zoho 侧已交付,平台侧待 B-1;C-3 被事实超越(流水线已运行);新增 series→SKU 映射、HMAC 盐协议、日期钳制三项修复。

## 一、项目定义与边界(不变)

基于约 1 万台在役变频热泵外机(R410A,3 系列 × 2 型号,208/230V 单相,美国全境,内外机无通讯 + 第三方内机混搭)的 10 秒级运行数据,构建面向售后与品质工程的可解释 FDD 体系:物理残差 + 统计漂移 + 可解释树模型,全流程本地运行,按类别分档放行。

明确不做:噪音类(无振动/声学通道);室内侧具体故障(只出非特异旗标);主板类建模(无热力前兆,平台故障码已覆盖,仅留 Tf/电压旁路筛查);深度时序模型(独立设备数不支持);遥测数据喂外部大模型(体量 + 安全双重排除)。

## 二、数据资产与锁定口径

四层数据:现场遥测(1 万台 × 10 秒 × 最长 10 年,48 列)/ 实验室(AHRI 210/240 稳态 + 大冷大热回油除霜极限工况;极限工况兼作瞬态标签训练稳态切分器)/ 产线(全量出厂联机,25–28℃;每台部件级出生指纹,**对充注状态无效**——安装改写系统级状态)/ 标签(平台实时故障事件 + Zoho 工单,本轮已量化,见 §4)。

**锁定口径十条**(做错任何一条下游全错):

1. Lp/Hp 为**密封式表压**,单位 bar,转绝压全机队固定 +1.013 bar,无海拔修正。
2. Sc 公式带 −1 经验偏置(Sc = Tsat(Hp) − Tl − 1),残差分析**必须剥掉 −1**。
3. Sh = Ts − Tsat(Lp),标准定义,测点在压缩机回气管进口;制热被伺服钉在近零,按变更记录第 2 条处理。
4. 制热 Sc 含安装管路温降偏置(变更记录第 3 条)。
5. TH = 室外盘管中部,控制用途化霜;诊断用途随模式翻转:制热 → 结霜/蒸发换热残差,制冷 → 冷凝侧诊断。Tl = 室外机液管(文档称"冷凝器出口")。
6. PowerComp(电功率)− PowerCompTheo(热功率)= 损耗 → 压缩机效率退化残差,文档落锚。
7. V1 = 208/230V 双额定(187–256V),机队双峰 → 电流类跨机残差必须以 V1 为协变量。
8. 控制器内置 9 个保护位(含 CoolantLackLimit),定位为超阈值安全保护;本项目做**保护触发前的早期残差趋势 + 现有规则漏报误报审计**,提前天数是第一个价值硬指标。
9. 能效/容量残差禁用 PowerIn(含曲轴加热带),用 PowerComp + 风机。
10. Tcs 为动态目标冷凝温度 → 学习/故障判据直读(变更记录第 4 条)。

**状态码字典**:AcState 全表与 St 已锁定(变更记录第 4 条);CompState:0 = 停机,1 = 正常压缩运行,2 = 特殊程序——其中真化霜签名 = St 翻转 + Th 冲 34~36℃ + 风机 ~300 + Exv ~220 + W 辅热;另有一类周期低频程序(St=1 不换向、Rps~33、Exv~75、风机~630、每 30–50 分钟一次、每次 ~9.5 分钟),名称待固件团队确认(附录 A),稳态分析中按剔除处理。

**24V 信号**:Y 制冷/压缩呼叫,W 辅助制热(有 Y)/紧急制热(无 Y),O 模式信号(0 = 制热/送风),G 内风机;化霜掉温触发温控器 W 呼叫为正常链条。

**低优先挂起**:时间戳时区口径(字段说明自标"需确认";对 0–14 天窗口匹配无影响)、N0、Cch、V12、V15、DayTime。

## 三、核心技术判断(不变,两处强化)

能力度量单位 = **带维修闭环、工况异质的独立故障设备数**。三档:5–10 台报方向;30–50 台报误报率(售后可用);50–100 台带完整前兆轨迹可做预测。低于 30 台的误报率是过拟合假数字。修后残差回归基线 = 自动闭环验证。

控制补偿把信号从状态量搬到执行器(Exv 同工况漂移 = 早期泄漏通道)——v1.1 起为实测事实(Sh 被钉零即证据)。学习漂移 vs 故障漂移:v1.1 起有 Tcs 直读判据 + 温控器信号统计双通道。

三层基线:每 SKU 实验室包络(6 个)/ 每 SN 出厂部件指纹(排除充注)/ 每台装机后 30 天现场冷媒基线(装机日用平台首条遥测日期做代理)+ 滚动同工况分箱自基线。传感器信任层先于一切故障诊断。方法栈:物理残差 → CUSUM/鲁棒 Z/变点 → XGBoost/RF 症状组合;交叉验证**必须按设备切分**(leave-units-out)。

**阶段 1|标签工程**——流水线已运行,进入修复 + 复核 + 连接三步:

4. **复核队列重排序**:现行 High 无区分度(硬件+有效SN 内 1,806/1,865 皆 High),500 行导出被 installation 类占 58%(291),冷媒类仅 18 条——重排权重 = 类别 FDD 价值 × SN 歧义(multi 优先)× 分类置信低(硬件类中位数 0.60)× 年份近。
5. **series→SKU 范围过滤**:硬件子集 34% 系列 Unknown,且出现 R410A 6-SKU 之外的产品线(南方热泵、侧出风 SDi)——需系列映射表,范围外打标保留不删。
6. 两文件编码不一致(candidates=GB18030,queue=UTF-8-BOM)→ 统一 UTF-8。

复核人力(D-2 具体化):top-3 类的 multi 解歧 = 605 + 253 + 226 ≈ 1,084 行,含脱敏文本每行约 30 秒 ≈ **1.5 人日**。Error Code 非空率 70.8%,解歧时可用工单故障码与 SN 候选交叉验证。

**阶段 2–6 不变**(物理信任层 → 三层基线 → 冷媒样板 + CoolantLackLimit 审计 → 标签放大 → 分类别放行),阶段 2 新增:CoolProp 替换查表、稳态切分器把 CompState=2 周期程序段全部剔除。

## 附录 A|致固件/控制团队:CompState 状态码确认(可直接转发)

背景:分析外机 10 秒遥测(样本 2023-03-20,制热,环温 −7~−1℃,5 小时)时需确认 CompState 官方定义。观察:CompState 取 0/1/2;0 = 压缩机停(转速 0),1 = 正常压缩运行,无疑义。CompState=2 出现 7 段,每段约 9–10 分钟,分两类:

- 2 段:四通阀换向(St→0),室外盘管中部 Th 从 −9~−13℃ 升至 +34~36℃,外风机降至约 300 rpm,EEV 开至约 220,W 辅热有输入——判断为化霜。
- 5 段:**不换向**(St 保持 1),Th 不升,压缩机降速至约 33 rps(正常约 67),EEV 收至约 75(正常约 148),外风机降至约 630 rpm(正常约 930),低压短时下探,每 30–50 分钟周期性出现。

问题 1:CompState 的官方枚举定义(0/1/2 各是什么,是否存在其他取值)?
问题 2:第二类周期程序是什么功能(回油运转 / 除霜需求判定 / 防冷风相关 / 其他)?触发与退出条件是什么?——决定该时段在稳态分析中剔除或保留。
问题 3:两次化霜中仅一次伴随 AcState=7(另一次 AcState 保持 5),AcState 置 7 的条件是什么?

答复三行文字即可。

## 附录 B|平台故障事件表·数据请求规格(可直接转发)

用途:作为标签时间脊柱,与已处理的 Zoho 工单(硬件类带有效 SN 1,865 张)按 SN + 时间窗连接,产出每类可训练标签集。

请求字段(每故障事件一行):
1. SN——建议平台侧用与 Zoho 侧**相同 HMAC 盐**直接提供 hash_sn(16 位),明文不出域,盐由数据管理方单独保管;
2. 事件时间戳(明确时区,建议 UTC + 当地时区两列);
3. 故障码原始值;
4. 故障码字典:码 → 名称 → 对应保护位(HpLimit/…/CoolantLackLimit)→ 严重级;
5. 事件恢复/清除时间戳(若有);
6. 机型号/系列;
7. **该 SN 的遥测覆盖区间**(首条与最新遥测日期)——判定工单标签是否有数据可用;
8. 固件版本(若有)。

连接协议:平台事件时间 ∈ [Zoho 工单事件日 − 21 天, + 3 天];验证键 = Zoho 工单 Error Code(非空率 70.8%)与平台故障码一致性抽检。

优先切片(若全量审批慢):冷媒类 668 个候选 hash_sn 名单可即刻提供,先拉这批的事件历史 + 遥测覆盖区间;连同同 SKU 同气候区对照约 1,400 台,构成阶段 0 第一批次。
