这是什么

FDD(北美住宅变频热泵外机故障诊断)系统的本地交互控制台:把管线的装载、物理转换、工况判定、 诊断矩阵、回归自检包装成可点击界面。零新增依赖(纯 Python 标准库 + 项目既有模块),仅绑定本机 127.0.0.1,不向任何外部服务发送数据 —— 与项目「全流程本地」安全承诺一致。

各页怎么用

  • ① 数据总览:点「装载实验室数据」(约 40–60s)。看四类信息:KPI 卡(行数/锚数/隔离数)、 覆盖门(每 SKU 须 ≥4 健康额定工况)、包络概览(各工况预测容量,注意 2436AA H2 为单行支撑)、 健康告警(已知事项 + 装载后动态项)。「不确定清单」列出需人工复核的段。
  • ② 单点诊断:输入残差(实测 − 该机基线)。典型用例——制冷早期少冷媒: sh_resid=0.5、sc_resid=−2.0、capacity_resid=−0.10 → refrigerant_low_or_leak,置信 0.65; 把 capacity_resid 改成 0 → 置信降为 0.5(sc 单证)。结果卡给出证据链、反证、判定过程逐步 解释与现场核查清单。
  • ③ 物理计算器:输入表压/温度读数,得 sh_phys/sc_phys/dsh_phys/饱和温度/tcs_gap 与 工况归属。用于现场快照核对与传感器合理性检查(负过热度/负过冷度会给出提示)。
  • ④ 文件体检:新拿到一个 RamChecker CSV,先体检再入库:是否 63 列方言、有无表外枚举值、 稳态率、能铸哪些工况。fault_injected 登记文件会被明确标注。
  • ⑤ 系统自检:一键跑 M0/M1/M2/规格套件,对照页面上的预期基线读结果。

读数与口径速查(容易踩的坑)

  • 压力是密封表压(bar),绝压 = 表压 + 1.013,全设备固定,禁海拔修正(铁律 1)。
  • 过冷度物理口径 sc_phys = tc_sat − Tl,没有 −1(控制器显示偏置已剥除,铁律 2)。
  • 固件上报 Sh/Sc 含系统性饱和表偏差(约 −2% 压力尺度),残差分析一律用本工具的物理口径。
  • 制热 Sh≈0 是正常伺服态,不是故障(铁律 3);制冷 EEV 满开是正常控制,exv_resid≡0(铁律 17)。
  • 诊断页输入的是残差不是读数:sh_resid 的参照面在制冷侧 = 逐机×同箱(DK-017), 跨机比较无效(实测同箱跨机散差 ~9K)。
  • severity 输出 fitted=False:是方向性代理值,欠充梯度数据到货后才有工程量纲。
  • dsh 安全参考带 5.0K 为暂定值,只产生核查清单注释,不参与判定。

故障排除

  • 页面打不开:确认控制台窗口在运行;端口被占用时用 --port 8899 重启。
  • 装载报错「目录不存在」:实验室数据须在 data/raw/lab/(含 監控数据 目录)。
  • 自检报 3 skipped:预期现象(holdout 等 M3、prodline×2 等盐协议),不是故障。
  • 计算器饱和温度为空:检查表压是否在查表范围(绝压 1–46 bar),超范围会明确警告。
  • 任何红色报错:均附「怎么办」提示;截图 + 控制台窗口堆栈即可反馈。
