# FDD 本地控制台 · 使用手册

版本:v2(2026-07-07)· 适配控制台 v2(含"运行数据检测与诊断")
本手册同时是页面「⑥ 使用说明」的正文来源(文件:`app/manual.md`,改这一份即两处同步)。

---

## 1. 系统是什么

FDD(北美住宅变频热泵外机故障诊断系统)的**本地交互控制台**:把命令行管线包装成可点击页面,覆盖
数据装载、单点诊断、物理量计算、运行数据检测与诊断、回归自检。

**架构与安全承诺**(与项目硬约束一致,使用者无须配置):

- 全流程本地:服务只绑定 `127.0.0.1`,不向任何外部服务发送任何数据;前端无 CDN 资源,断网可用。
- 零新增依赖:后端为 Python 标准库,分析能力全部来自项目既有管线模块(c4/conv/seg/diag/sense/baseline)。
- 只读消费:控制台**不修改任何管线代码、不改任何数据文件**;上传文件只进本机临时目录,不进 data/ 仓库。
- 阈值单一真相:界面显示的所有判定阈值实时读自 `config/calibration.yaml`,标定重估后界面自动跟随。

## 2. 启动与停止

| 操作 | 方法 |
|---|---|
| 启动 | 双击 fdd 根目录 `start_console.bat`(自动开浏览器),或 `.venv/Scripts/python app/fdd_console.py` |
| 换端口 | `start_console.bat --port 8899`(默认 8765) |
| 不自动开浏览器 | 加 `--no-browser` |
| 停止 | 关闭命令行窗口,或窗口内 Ctrl+C |

启动后地址:`http://127.0.0.1:8765/`。命令行窗口保持开启即服务在线。

## 3. 推荐工作流

**新数据文件到手(最常用)**:
④ 运行数据检测 → 选取文件 → 「快速体检」看数据质量 → (已装载基线后)「检测与诊断」→
读段级结论与文件级判定 → 需要人工深究的段,把段残差抄进 ② 单点诊断做敏感性试验
(例如把某个残差归零看结论变不变)。

**现场读数快速核对**:
③ 物理计算器输入表压/温度 → 看 sh/sc/dsh 与工况归属是否合理 → 异常读数会有提示。

**系统状态巡检**:
① 数据总览装载 → 看覆盖门/包络/告警 → ⑤ 系统自检跑 M0/M1/M2 对照预期基线。

## 4. 各页详解

### ① 数据总览

**做什么**:装载 `data/raw/lab/` 全部实验室数据(完整 C4 管线:认证窗切片 → 10 秒时基归一 →
稳态/锚分段 → 故障注入分流 → envelope_input 派生 → DK-016 隔离标注),缓存于内存供全站使用。

**输出解读**:

| 区块 | 含义 |
|---|---|
| KPI 卡 | 载入行(10s 面)/ 机台清单 / 额定锚行(rating_anchor)/ 拟合输入行(envelope_input)/ 故障注入行 / 隔离标注行 |
| 覆盖门 | 每 SKU 健康额定工况数,**≥4 达标**(绿),不足标红 |
| 包络概览 | 各工况预测容量(kW)与 Ta 斜率(kW/K);拟合输入 = envelope_input 面 |
| 数据构成 | 机台 × 工况 × 类别 × 数据类型的行数/锚数明细 |
| 不确定清单 | AMBIGUOUS / UNKNOWN_CONDITION / UNSTABLE 段,供人工复核 |
| 健康与已知事项 | 常驻登记已知系统状态(见 §7),避免把已知态误读为新故障 |

注意:**装载是"检测与诊断"的前置**——诊断基线(逐机健康箱中位)从这份数据构建。

### ② 单点诊断

**做什么**:手动输入一组**残差**(= 实测 − 该机基线,不是绝对读数),运行 M-DIAG 规则矩阵,
输出 C5 五件套:故障假设 / 证据 / 反证 / 置信度 / 现场核查清单,并逐步解释每个判据的比较过程。

**字段含义**:

| 字段 | 含义 | 单位 |
|---|---|---|
| sh_resid | 过热度残差(制冷参照面 = 逐机×同箱,DK-017) | K |
| sc_resid | 过冷度残差(少冷媒早期主通道) | K |
| capacity_resid | 容量残差,相对值(−0.10 = 比基线低 10%) | 比例 |
| exv_resid | EEV 开度残差 | 步 |
| dsh_phys | 排气过热度(绝对值,可留空 = 无测点) | K |
| EEV 饱和 | 制热 Sh 证据启用门(铁律 3);制冷不消费 | — |

**制冷判定矩阵**(DK-015 次序:先过热度后过冷度;阈值实时读配置):

| SSH(sh_resid) | SC(sc_resid) | 结论 | 置信 |
|---|---|---|---|
| 正常带(\|sh\|≤3.0K) | < −1.5K | 少冷媒/泄漏(早期) | sc 单证 0.5;sc+容量双证 0.65 |
| > +3.0K | < −1.5K | 少冷媒/泄漏(中晚期,Sh 自门控) | 0.7 |
| > +3.0K | > +1.5K | 节流受限(SC 正向自动写入对少冷媒的反证) | 0.65 |
| 其它(含 exv 单独偏离) | — | none | 0.1 |

制冷侧 **exv_resid 永不作泄漏证据**(铁律 17:EEV 满开非伺服);单独大幅偏离 = EEV 卡滞/控制
异常指示,界面会单独提示。

**制热判定**(铁律 3):泄漏 = EXV↑(≥10)+ SC↓(≤−0.8)+ 容量↓(≤−0.05)三特征同触;
只有容量低而冷媒侧干净 → 内机侧非特异;Sh 证据仅在 EEV 饱和后启用。

**结果解读**:置信度为 v0 占位标度(M4 换模型分);severity 为未拟合方向性代理
(fitted=False),仅供相对参考;核查清单是给现场工程师的动作项。

### ③ 物理计算器

**做什么**:输入一组原始读数快照,按 C1 契约物理口径计算派生量并判工况归属。

**口径要点**(易错):压力输入为**密封表压 bar**,绝压 = 表压 + 1.013,禁海拔修正(铁律 1);
过冷度 sc_phys = tc_sat − Tl,**无 −1 显示偏置**(铁律 2);饱和温度用 CoolProp R410A 查表
(te_sat 露点 / tc_sat 泡点),固件上报 Sh/Sc 有系统性偏差,残差分析一律用本口径。

**输出**:lp_abs/hp_abs、te_sat/tc_sat、sh_phys/sc_phys/dsh_phys、tcs_gap(给 Ta 加判工况字母
与容量档;给 Tcs 加算目标追踪差)。异常读数自动提示:负过热度(回液风险/测点错)、负过冷度、
排气过热度低于安全参考带(5.0K,暂定)、压力超查表范围(绝压 1–46 bar)。

### ④ 运行数据检测与诊断

**做什么**:对单个 RamChecker 监控 CSV 做两级分析——

- **快速体检**(无须装载数据):时间轴重建 → 枚举隔离 → 10s 重采样 → 稳态/锚分段 → 逐行工况
  归属。输出行数、时距、Ta/模式/稳态率、锚统计、工况分布、表外枚举值、问题清单。
- **检测与诊断**(须先在①装载数据):体检基础上 →
  1. 按稳态段切分(段 ≥ 2 分钟);
  2. 每段计算物理量中位(sh/sc/dsh/容量/Exv/Ta/频率)并判工况;
  3. 从健康池按**逐机×同箱**(机台 × 模式 × Ta 2K 箱 × 频率 10Hz 档)取基线中位,算段级残差;
     精确箱无基线时放宽到同 Ta 箱全频段(结果标注"跨频率档参照");
  4. 段残差喂 M-DIAG → 段级 C5 诊断(证据/置信/清单);
  5. 传感器信任检验(sense):以该机健康数据拟合箱式参照,检验本文件 Ts/Tl/Lp/Hp/Th/Ta;
  6. 汇总文件级结论(最严重假设 + 置信 + 各段计数)。

**文件选取方式(三选一)**:

| 方式 | 适用 | 说明 |
|---|---|---|
| 浏览实验室数据 | data/raw/lab 内文件 | 目录树点选,自动识别机台与数据类型 |
| 从本机选取上传 | 任意位置文件 | 浏览器选取 → 传入本机临时目录分析(不进仓库);机台需手动选择 |
| 手动粘贴路径 | 高级用法 | 折叠在"高级"里,兼容旧习惯 |

**基线规则(为什么有的段"不可判")**:诊断残差的参照面是**逐机**的
(制冷 sh 参照面 = 逐机×同箱,DK-017;制热 Sc 禁跨机绝对比较,铁律 10)。
若该机台在健康池中没有对应工况箱的数据,系统**不会**拿别的机台凑基线,而是明确标注
"无逐机基线,不可判",并列出可算的通道观察。这是刻意设计:宁缺勿错。

**结果解读**:

| 输出 | 含义 |
|---|---|
| 文件级结论卡 | 全文件最严重假设与置信;各段假设计数;传感器旗提示 |
| 传感器信任表 | ok / flagged / no_reference;flagged 时提示"并发时故障解释优先,传感器复核并行"(锁定发现:信任旗为仲裁输入非独立判决) |
| 稳态段表 | 每段:时间/模式/工况/物理量中位/残差/假设徽章/置信;点开看完整 C5(证据链、判定过程、核查清单、基线箱信息) |

**典型验证案例**(可自行复现):31 号机 `RamChecker_20231127123005.csv`(健康 H1)检测应
以 none 为主;`RamChecker_20231209162220.csv`(受控 50% 欠充注入)检测应在 H1 段给出
refrigerant_low_or_leak(sc_resid 显著负)。

**批量检验(设备级故障检测)**:选中**目录**(目录树中点「[批量,N 个 CSV]」或「批量检验当前
目录」)或**多选上传**多个文件后,「检测与诊断」按钮自动进入批量模式:

- 逐文件跑完整检测诊断链(后台任务,实时进度);单文件失败不中断整批,错误入明细行;
- **设备级汇总**:按机台聚合——文件数 / 异常数 / 不可判数 / 传感器旗文件数、**最严重发现**
  (假设+置信+文件+时间)、段级假设计数、**时间线**(按文件时间锚排序的结论徽章序列,
  可直观看到故障演进,如 31 号机 11 月健康 → 12-09 欠充文件集中变红);
- 文件明细表点击任意行展开该文件的完整段级诊断;
- 「导出 JSON 报告 / CSV 摘要」下载留档(CSV 带 BOM,Excel 直接打开不乱码)。

批量注意事项:目录模式递归包含子目录全部 CSV(上限 500,超出请分批);lab 目录内机台自动
识别,「机台」下拉留「自动」;上传批量无目录上下文,须在下拉框选机台(整批同机台,混装请分批)。

### ⑤ 系统自检

在项目虚拟环境跑 pytest 回归套件(子进程剔除 FDD_HMAC_KEY):

| 套件 | 覆盖 | 预期基线 | 时长 |
|---|---|---|---|
| M0 | conv/seg/zoho | 13 passed | ~30s |
| M1 | feat/drift/sense/label/valid/diag | 53 passed | ~25s |
| M2 | c4/baseline/envelope/SSD | 55 passed + 3 skipped | ~60s |
| 规格 | condition_of + envelope_input | 49 passed | ~5s |

**3 skipped 是已知等待项**(holdout 等 M3 现场数据;prodline×2 等 HMAC 盐协议),不是故障。
任何 **failed** 都值得回报。

## 5. 判定阈值速查(与 calibration.yaml 同源)

| 常量 | 值 | 说明 |
|---|---|---|
| sh_norm_band_cool | ±3.0 K | 制冷 SSH 正常带(参照面 per_unit_bin,≈4σ) |
| sh_high_cool | +3.0 K | 制冷 SSH 高界(带边即界) |
| sc_low / sc_high | −1.5 / +1.5 K | 制冷 SC 低阈(少冷媒)/ 高阈(节流受限) |
| cap_low | −0.07 | 制冷容量佐证阈(非硬门) |
| dsh_safety_min_k | 5.0 K(暂定) | 排气过热度安全参考带,仅注释不判定 |
| 制热 EXV_UP / SC_DOWN / CAP_DOWN / SH_HIGH | +10 / −0.8 / −0.05 / +2.0 | 制热三特征与 Sh 门 |
| 工况容差 | ±1.5 ℃ | 字母归属(最近中心);H0/NONSTD 重叠区由最近中心裁决 |
| 全负荷频率带 | 制冷 [66,76] / 制热 [70,86] Hz | lab InvHz 口径,闭区间,带外=UNKNOWN_CAPACITY |

## 6. 数据与口径速查(易踩的坑)

- 压力 = 密封**表压**(bar);绝压 = 表压 + 1.013;禁海拔修正。
- sc_phys 无 −1;固件 Sh/Sc 含约 −2% 压力尺度饱和表偏差,不要与物理口径混用。
- 制热 Sh≈0 是伺服正常态(铁律 3);制冷 EEV 满开是量产正常控制(铁律 17)。
- 制热 Sc 含安装管路温降偏置 → 只可逐机比较(铁律 10)。
- 残差的参照面是逐机的;跨机比较过热度在制冷侧实测散差 ~9K(结构性),无效。
- 控制器容量值(Qc/Qh)相对台架实测 +2%~+12%(均值≈+7%):逐机漂移免疫,绝对值须带修正。
- CompState=2(化霜/周期程序)全段剔除,不入稳态分析(铁律 6)。
- severity 输出 fitted=False = 未拟合方向性代理,等欠充梯度数据。

## 8. 故障排除(FAQ)

| 现象 | 处置 |
|---|---|
| 双击 bat 一闪而过 | 已修复(v2 起 bat 纯 ASCII);若复现,在 cmd 中手动运行看报错 |
| 页面打不开 | 确认命令行窗口在运行;端口占用时 `--port 8899` 重启 |
| 「数据尚未装载」 | 到①点「装载实验室数据」(约 20–60s) |
| 装载报"目录不存在" | 实验室数据须在 `data/raw/lab/`(含 监控数据 目录) |
| 检测报"该机台无健康基线" | 逐机参照面的刻意保护;补充该机健康数据后重装载 |
| 段标"无逐机基线,不可判" | 该机在此工况箱无健康数据;看"可算通道观察",勿硬下结论 |
| 上传后机台显示"未知" | 上传文件无目录上下文,请在下拉框手选机台再检测 |
| 计算器饱和温度为空/警告 | 表压超查表范围(绝压 1–46 bar),先核对读数 |
| 自检出现 failed | 截图输出回报;3 skipped 除外(预期) |
| 枚举隔离行很多 | 停机/过渡族(0/1/2/3/5/10)为预期;**表外值**(如 12/8/65/66/负值)已列厂商问题清单 |

## 9. 术语表

| 术语 | 含义 |
|---|---|
| C5 | 诊断输出契约:{假设, 证据[], 反证[], 置信度, 现场核查清单[], severity} |
| 残差(resid) | 实测 − 基线;诊断的输入口径 |
| 逐机×同箱 | 基线参照面:机台 × 模式 × Ta 2K 箱 × 频率 10Hz 档 |
| 锚(anchor) | 稳态基准行:clean_steady(净盘管)/ frosting_steady(结霜准平衡) |
| rating_anchor / envelope_input | 额定覆盖门用 / L1 包络拟合输入(NONSTD 规则①②拆分) |
| fault_injected | 受控故障注入数据:保留加载但排除出一切健康基线 |
| cooling_ref_quarantine | DK-016 机台级异常隔离旗(u31×A) |
| 工况字母 | AHRI 210/240-2026:A/B/C_or_D(制冷)、H0–H4/NONSTD_HEAT(制热) |
| sense 信任旗 | 传感器一致性检验输出(ok/flagged/no_reference);仲裁输入非独立判决 |
